# 英文 Nintendo 类 SEO 站建站流程

## 一、项目初始化

```bash
npx create-next-app@latest my-site --typescript --tailwind --app
cd my-site
pnpm install
```

关键配置文件：
- `next.config.ts` — 图片域名白名单等
- `app/layout.tsx` — 全局 metadata、script 注入点
- `app/globals.css` — 全局样式

---

## 二、域名 & 部署平台

1. 购买域名（Cloudflare 购买最省事，DNS 自动托管）
2. 选择部署平台：
   - **Cloudflare Workers**（本站方案）：`pnpm add -D @opennextjs/cloudflare wrangler`，配置 `wrangler.toml`
   - 备选：Vercel（更简单，免费额度够用）

---

## 三、页面结构规划

按关键词规划 URL 树，优先级顺序：

```
/ (首页，最高优先级)
/[核心关键词] (主要分类页)
/[核心关键词]/[长尾词] (子页面)
```

本站例子：`/tomodachi-life-personalities` → `/tomodachi-life-living-the-dream` → `/tomodachi-life-living-the-dream/pc`

---

## 四、SEO 配置

**`app/layout.tsx` metadata：**

```tsx
export const metadata: Metadata = {
  title: '...',
  description: '...',
  keywords: [...],
  metadataBase: new URL('https://www.your-domain.com'),
  openGraph: { ... },
  twitter: { card: 'summary_large_image', ... },
  robots: { index: true, follow: true },
  other: { 'google-adsense-account': 'ca-pub-XXXXXXXX' },
};
```

**`app/sitemap.ts`：**
- 列出所有 URL，设置 `priority` 和 `changeFrequency`
- 首页 `priority: 1.0`，核心页 `0.9`，子页 `0.8`，工具页 `0.6`

**`app/robots.ts`：**

```ts
export default function robots() {
  return {
    rules: { userAgent: '*', allow: '/' },
    sitemap: 'https://www.your-domain.com/sitemap.xml',
  };
}
```

---

## 五、内部链接

创建 `components/RelatedPages.tsx`，在每个页面底部双向引用相关页面。规则：
- 每页至少链接 3 个相关页面
- 链接要**双向**（A 链接 B，B 也链接 A）
- anchor text 使用目标页的核心关键词

---

## 六、Google Analytics 接入

在 `app/layout.tsx` 的 `<head>` 中用 `next/script`：

```tsx
<Script
  src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"
  strategy="afterInteractive"
/>
<Script id="google-analytics" strategy="afterInteractive">{`
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'G-XXXXXXXXXX');
`}</Script>
```

Measurement ID 在 GA4 后台 → 数据流 → 网页流详情里找。

---

## 七、Google AdSense 接入

两处添加（均在 `app/layout.tsx`）：

```tsx
// 1. metadata.other 中添加 meta 标签
other: { 'google-adsense-account': 'ca-pub-XXXXXXXXXXXXXXXX' }

// 2. <head> 中注入脚本
<Script
  src="https://pagead2.googlesyndication.com/pagead/js/adsbygoogle.js?client=ca-pub-XXXXXXXXXXXXXXXX"
  strategy="afterInteractive"
  crossOrigin="anonymous"
/>
```

部署后去 AdSense 后台手动「+ 新增网站」，Google 自动验证脚本。

---

## 八、部署

```bash
# Cloudflare Workers
pnpm build
pnpm opennextjs-cloudflare build
pnpm wrangler deploy
```

---

## 九、Google Search Console

1. 添加站点（`sc-domain:your-domain.com` 或 URL 前缀）
2. 验证所有权（Cloudflare DNS 添加 TXT 记录最方便）
3. 提交站点地图：`https://www.your-domain.com/sitemap.xml`

---

## MCP 能做什么（上线后维护阶段）

> MCP 适合**上线后的数据分析和运营决策**，建站阶段用处不大。

| MCP | 能做的事 |
|-----|---------|
| **Google Search Console** | 提交/检查站点地图、查看关键词排名、检查页面索引状态、发现快速提升机会 |
| **Google Analytics** | 查询流量报告、分析用户行为、对比时间段数据 |
| **Google AdSense** | 查看收入报告、列出广告单元、查看政策问题 |
