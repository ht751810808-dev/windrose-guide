# Tomodachi Life — 产品需求文档 (PRD)

> 版本：v1.0 | 日期：2026-05-08 | 技术栈：Next.js 15 · App Router · TypeScript · Tailwind CSS

## 一、项目概述

**项目名称：** Tomodachi Life 粉丝站  
**目标：** 围绕关键词 "Tomodachi Life" 建设一个 SEO 驱动的英文内容站，覆盖游戏攻略、ROM/NSP 资源导航、Mii QR 码、性格系统等核心搜索需求，通过 16 个子页面最大化关键词覆盖面，形成站内链接网络。  
**域名：** 待定（需在 `app/sitemap.ts` 和 `app/layout.tsx` 的 `metadataBase` 中统一替换）

## 二、全局 SEO 内容规范（所有页面强制执行）

### 2.1 词数要求
- **每个页面正文词数 ≥ 1500 words**（落地页 `/` 建议 ≥ 2000 words）
- 词数统计范围：正文可见文本（H1/H2/H3/p/li），不含导航、Footer、代码注释

### 2.2 主关键词密度
- **每个页面的主攻关键词出现频率 ≥ 3%**（以该页面总词数为基数）
- 示例：1500 词页面中主关键词至少出现 **45 次**
- 关键词变体（复数、带连字符）计入统计，但避免堆砌，保持自然语义

### 2.3 标题层级规范（H1 / H2 / H3）

| 层级 | 数量 | 规范 |
|------|------|------|
| H1 | 1 个/页（唯一） | 必须包含该页面主攻关键词，位于页面顶部 Hero 区 |
| H2 | ≥ 4 个/页 | 至少 3 个 H2 包含主攻关键词或其近义变体 |
| H3 | ≥ 6 个/页 | 覆盖长尾词、问题型搜索词、功能小节标题 |

### 2.4 Meta 信息规范（每页独立）

```tsx
export const metadata: Metadata = {
  title: '{主关键词} – {差异化描述} | {站名}',   // ≤ 60 字符
  description: '{含主关键词，自然句，150–160字符}',
  keywords: ['{主关键词}', '{长尾词1}', '{长尾词2}', ...],
  alternates: { canonical: 'https://{domain}/{slug}' },
  openGraph: { ... },   // 每页独立 OG title/description
};
```

### 2.5 结构化数据
- 落地页 `/`：`FAQPage` + `WebSite` schema
- 攻略/指南页：`Article` schema（含 `datePublished`, `dateModified`）
- 工具页（QR Codes, Save Editor）：`SoftwareApplication` schema
- 所有页面：`BreadcrumbList` schema

### 2.6 内链规范
- 每个子页面至少 **3 条内链** 指向相关子页面
- 落地页 `/` 必须链接到所有 P1 子页面
- `Living the Dream` Hub 页必须链接到其 5 个子页
- 锚文本必须使用目标页的主关键词（不用"点击这里"）
